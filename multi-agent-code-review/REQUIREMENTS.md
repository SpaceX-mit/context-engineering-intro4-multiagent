# Multi-Agent Project Development System

基于 Multi-Agent Code Review 项目，打造一个完整的 AI 驱动项目开发系统。

## 1. 项目概述

### 1.1 目标
构建一个类似 Codex Desktop 的桌面应用，让多个专业 AI Agent 协作完成完整项目开发。

### 1.2 核心价值
- **多 Agent 协作**: 5+ 专业 Agent 分工明确
- **端到端开发**: 从需求到可运行代码
- **质量保障**: 自动审查、测试、修复
- **可视化流程**: 实时展示 Agent 工作状态

---

## 2. Agent 架构

### 2.1 Agent 角色定义

| Agent | 职责 | 核心能力 |
|-------|------|----------|
| **Coordinator** | 任务分解、协调、结果汇总 | 理解需求、任务分配、流程控制 |
| **Planner** | 项目规划、任务拆解 | 分析需求、制定计划、估算工时 |
| **Coder** | 代码生成、重构 | 实现功能、遵循规范、代码质量 |
| **Reviewer** | 代码审查、质量评估 | 逻辑检查、安全扫描、性能分析 |
| **Linter** | 代码风格、格式化 | 语法检查、风格统一、自动化格式化 |
| **Tester** | 测试生成、覆盖率分析 | 单元测试、集成测试、测试覆盖 |
| **Fixer** | 问题修复、代码优化 | 自动修复、迭代优化 |

### 2.2 Agent 协作流程

```
用户需求
    │
    ▼
┌─────────────┐
│ Coordinator │ ←── 接收需求，理解任务
└──────┬──────┘
       │ 分解任务
       ▼
┌─────────────┐
│   Planner   │ ←── 制定实施计划
└──────┬──────┘
       │ 输出计划
       ▼
┌─────────────┐     ┌─────────────┐
│   Coder     │ ←── │   同时进行  │
└──────┬──────┘     │  Linter    │
       │             └──────┬──────┘
       ▼                   │
┌─────────────┐            │
│  Reviewer   │ ←──────────┘
└──────┬──────┘
       │ 发现问题
       ▼
┌─────────────┐
│   Fixer     │ ←── 自动/手动修复
└──────┬──────┘
       │ 修复完成
       ▼
┌─────────────┐
│  Tester     │ ←── 生成测试
└──────┬──────┘
       │ 测试通过
       ▼
    项目完成
```

### 2.3 迭代优化流程

```
Review → Fix → Review (循环直到通过)
         ↓
      人工确认 (可选)
         ↓
      进入下一任务
```

## 3. 功能说明书

### 3.1 核心功能

#### 3.1.1 需求管理
- **需求输入**: 文本描述、文件导入、语音输入
- **需求解析**: 自动提取关键技术点
- **需求确认**: 用户确认后再开始开发
- **历史记录**: 保存和回溯历史需求

#### 3.1.2 项目管理
- **项目创建**: 从模板或空白开始
- **项目结构**: 自动生成目录结构
- **文件管理**: 创建、编辑、删除、重命名
- **多项目**: 支持同时管理多个项目

#### 3.1.3 Agent 协作
- **并行执行**: Linter 和 Reviewer 同时运行
- **顺序执行**: Planner → Coder → Reviewer → Fixer
- **迭代优化**: Review → Fix 循环直到通过
- **人工干预**: 关键节点可人工确认

#### 3.1.4 代码审查
- **Linter 检查**: 语法、风格、格式化
- **Reviewer 评估**: 逻辑、安全、性能
- **Security 扫描**: 漏洞检测
- **复杂度分析**: 代码复杂度报告

#### 3.1.5 测试生成
- **单元测试**: 为每个函数生成测试
- **集成测试**: 测试组件交互
- **覆盖率报告**: 显示测试覆盖百分比
- **失败重试**: 测试失败自动重试

#### 3.1.6 自动修复
- **Linter 修复**: 自动格式化
- **Reviewer 修复**: 建议修复方案
- **一键修复**: 执行所有可自动修复的问题
- **修复确认**: 修复前需用户确认

### 3.2 Agent 输出格式

所有 Agent 输出必须符合以下 JSON 格式：

```json
{
  "status": "success|error|pending",
  "agent": "agent_name",
  "results": [...],
  "next_action": "recommend_action"
}
```

### 3.3 工作流类型

| 工作流 | 描述 | 适用场景 |
|--------|------|----------|
| `sequential` | 顺序执行 | 简单任务、新手项目 |
| `concurrent` | 并行执行 | 复杂任务、提速 |
| `iterative` | 迭代优化 | 高质量要求 |

### 4.4 交互流程

#### 新建项目流程
```
1. 点击 [New Project]
2. 输入项目名称
3. 选择模板 (可选)
4. 确认 → 创建项目结构
5. 输入需求描述
6. 点击 [Start Development]
7. Agent 开始协作开发
```

#### 代码审查流程
```
1. 选择文件
2. 点击 [Review]
3. Agent 面板显示运行状态
4. Terminal 显示审查过程
5. 问题高亮显示
6. 选择 [Auto-Fix] 或 [Manual Fix]
7. 重新审查直到通过
```

## 6. 安装与配置

### 6.1 环境要求

- Python 3.10+
- Ollama (本地运行 LLM)
- 8GB+ RAM
- 10GB+ 磁盘空间

### 6.2 安装步骤

```bash
# 1. 克隆项目
git clone <repo-url>
cd multi-agent-code-review

# 2. 创建虚拟环境
python -m venv venv
source venv/bin/activate  # Linux/Mac
# or
.\venv\Scripts\activate   # Windows

# 3. 安装依赖
pip install -r requirements.txt

# 4. 配置 Ollama
ollama pull llama3.2

# 5. 启动应用
python -m webui.app

# 6. 打开浏览器
# http://localhost:7860
```

### 6.3 配置项 (.env)

```env
LLM_PROVIDER=ollama
LLM_MODEL=llama3.2
OLLAMA_BASE_URL=http://localhost:11434
MAX_ITERATIONS=3
SANDBOX_ENABLED=true
```

## 7. 使用示例

### 7.1 开发一个新功能

```
用户输入:
"创建一个待办事项管理应用，支持添加、删除、完成任务"

系统执行:
1. Coordinator 解析需求
2. Planner 制定计划 (3个文件: models.py, views.py, app.py)
3. Coder 生成代码
4. Linter 检查风格
5. Reviewer 审查质量
6. Tester 生成测试
7. Fixer 修复问题
8. 输出完整项目
```

### 7.2 审查现有代码

```
用户选择: src/app.py
点击: [Review]

系统执行:
1. Linter 检查风格 → 发现 3 处
2. Reviewer 审查质量 → 发现 2 处
3. Security 扫描 → 无问题
4. Tester 检查测试 → 缺少测试

输出:
- 问题列表 (严重程度分级)
- 建议修复方案
- 一键修复按钮
```
